# Thorium Timetable - Technical Debt & Algorithm Improvements

## Code Quality Issues

1. **Layer violation**: `thorium-ui` directly depends on `thorium-infrastructure`
   - Fix: Create bootstrap interface in `thorium-application`, have UI depend only on that

2. **No connection pooling**: `SQLiteConnectionProvider.close()` is a no-op
   - Fix: Implement proper connection tracking and cleanup

3. **Minimal test coverage**: Only 1 unit test + 1 UI smoke test; scheduling engine has zero tests
   - Fix: Add unit tests for GreedyScheduler, BacktrackingScheduler, SoftConstraintScorer

4. **Duplicate code**: `TabuSearchStrategy` and `HybridMetaheuristicStrategy` each define their own `TabuList` and `NeighborMove`
   - Fix: Extract shared `TabuList` and `SchedulingMove` into a common utility class

5. **Dead code**: `SchedulingException` is defined but never thrown
   - Fix: Use it in generators and schedulers for meaningful error reporting

6. **Hardcoded soft constraint weights**: `SPREAD_WEIGHT=0.55` / `CONSECUTIVE_WEIGHT=0.45`
   - Fix: Make weights configurable via constructor/configuration

7. **No cancellation support**: `Task.cancel()` interrupts thread but generators don't check
   - Fix: Add `isCancelled()` callback interface and check points in generation loop

8. ~~**Missing port interface**: `SchoolSettingsRepository` port exists (false positive)~~
   - Status: Interface exists at `thorium-application/port/SchoolSettingsRepository.java`

9. **Destructive migration V9**: Drops old tables mid-migration — data loss risk
   - Fix: Wrap in transactional fallback with backup table

10. **Incomplete entity**: `Teacher` ignores `maxLessonsPerDay`/`maxLessonsPerWeek` from DB
    - Fix: Add fields to Teacher entity, use them in scheduling constraints

## Algorithm Improvements (Issue #11)

11. **Advanced scheduling algorithm improvements**
